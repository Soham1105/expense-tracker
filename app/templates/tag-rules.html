How it works

  • Rules run against all transactions when you click Apply Rules Now.
  • A transaction already tagged with the same tag is skipped (no duplicates).
  • Contains — case-insensitive substring match (most common).
  • Exact — full case-insensitive equality.
  • Regex — PostgreSQL-style regex (~*).
  • Transactions tagged by rules show as "auto" in Reports; manually tagged ones show as "manual".
